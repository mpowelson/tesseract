\doxysubsection{Detailed Description}
Represents material information extracted from a mesh file. 

\mbox{\hyperlink{classtesseract__geometry_1_1Mesh}{Mesh}} files contain material information. The mesh parser will extract the material information and store it in a \mbox{\hyperlink{classtesseract__geometry_1_1MeshMaterial}{Mesh\+Material}} instance. The \mbox{\hyperlink{classtesseract__geometry_1_1MeshMaterial}{Mesh\+Material}} class uses a subset PBR Metallic workflow, as specified in the gl\+TF 2.\+0 file format standard. The four parameters supported are base\+Color\+Factor, metallic\+Factor, roughness\+Factor, and emmisive\+Factor. (The \mbox{\hyperlink{classtesseract__geometry_1_1MeshTexture}{Mesh\+Texture}} class stores diffuse textures that can be used for decals and fiducial marks, and is stored separately from \mbox{\hyperlink{classtesseract__geometry_1_1MeshMaterial}{Mesh\+Material}}.) These four parameters and \mbox{\hyperlink{classtesseract__geometry_1_1MeshTexture}{Mesh\+Texture}} should be enough to display \char`\"{}\+CAD quality\char`\"{} renderings in visualizers. The full mesh file should be used when higher quality rendering is required.

The \mbox{\hyperlink{classtesseract__geometry_1_1MeshMaterial}{Mesh\+Material}} favors PBR materials extracted from gl\+TF 2.\+0 files. COLLADA does not support PBR. Only \char`\"{}\+Diffuse\char`\"{} and \char`\"{}\+Emissive\char`\"{} are read. \char`\"{}\+Specular\char`\"{} and \char`\"{}\+Ambient\char`\"{} are ignored.
